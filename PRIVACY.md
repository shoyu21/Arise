# Privacy Policy & Data Protection Statement

**ARISE - Awakening & Respiratory Integration for Safe Extubation**

Last Updated: December 2024

---

## Summary

**ARISE does NOT collect, store, or transmit any personal or patient data.**

---

## Data Processing Architecture

### Client-Side Processing Only

ARISE is designed as a **fully client-side application**. This means:

- ✅ All calculations occur locally in your web browser
- ✅ All data entered remains on your device only
- ✅ No data is transmitted to any external server
- ✅ No database connections exist
- ✅ No backend server processes your inputs

### Technical Implementation

```
User Input → Browser JavaScript → Local Calculation → Display Results
         (No external data transmission at any point)
```

## Healthcare Provider Responsibilities

While ARISE does not collect or store data, healthcare providers using ARISE should:

1. **Screen Sharing/Screenshots**
   - Be mindful when taking screenshots or sharing screens
   - Avoid capturing patient-identifiable information in screenshots

2. **Institutional Policies**
   - Follow your institution's data protection policies
   - Comply with local healthcare privacy regulations

3. **Physical Security**
   - Be aware of your surroundings when using ARISE at the bedside
   - Ensure patient information is not visible to unauthorized individuals

## Data Security Measures

Although ARISE does not collect data, the following security measures are in place:

1. **HTTPS Encryption**
   - All connections use HTTPS encryption
   - Data in transit is protected

2. **No Persistent Storage**
   - No cookies are set
   - No local storage is used
   - No session storage is used

3. **No External API Calls**
   - ARISE does not call external APIs with user data
   - All functionality is self-contained
